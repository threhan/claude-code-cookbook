## AI 写作检查

检测 AI 生成文章中的机械模式，并提出改进建议，使其更自然地道的中文。

### 使用方法

```bash
/ai-writing-check [选项]
```

### 选项

- 无：分析当前文件或选定的文本
- `--file <path>`：分析特定文件
- `--dir <path>`：批量分析目录中的文件
- `--severity <level>`：检测级别（all/high/medium）
- `--fix`：自动修复检测到的模式

### 基本示例

```bash
# 检查文件的 AI 感
cat README.md
/ai-writing-check
“请检查本文档的 AI 感，并提出改进建议”

# 分析特定文件
/ai-writing-check --file docs/guide.md
“请检测 AI 风格的表达方式，并提出自然的修改建议”

# 扫描整个项目
/ai-writing-check --dir . --severity high
“请仅报告项目中重要的 AI 感问题”
```

### 检测模式

#### 1. 列表格式的机械模式

```markdown
检测示例：

- **重要**：这是一个重要的项目
- 已完成的项目（带复选标记）
- 热门话题（带火焰表情符号）
- 准备开始（带火箭表情符号）

改进示例：

- 重要项目：这是一个重要的项目
- 已完成的项目
- 热门话题
- 准备开始
```

#### 2. 夸张/炒作的表达

```markdown
检测示例：
革命性的技术将改变行业。
这将彻底解决问题。
它像魔法一样工作。

改进示例：
有效的技术将为行业带来变革。
它解决了许多问题。
它运行顺畅。
```

#### 3. 机械的强调模式

```markdown
检测示例：
**想法**：我有一个新建议（带灯泡表情符号）
**注意**：重要警告（带警告表情符号）

改进示例：
想法：我有一个新建议
注意：重要警告
```

#### 4. 冗长的技术写作

```markdown
检测示例：
首先，进行设置。
您可以使用此工具。
性能将得到显着提升。

改进示例：
首先进行设置。
您可以使用此工具。
性能提升 30%。
```

### 与 Claude 协作

```bash
# 分析整个文档的 AI 感
cat article.md
/ai-writing-check
“请从以下角度分析并提出改进建议：
1. 检测机械表达
2. 提出自然的中文修改建议
3. 按优先级列出改进清单”

# 关注特定模式
/ai-writing-check --file blog.md
“请特别关注夸张和冗长的表达，并提出改进建议”

# 批量检查多个文件
find . -name "*.md" -type f
/ai-writing-check --dir docs/
“请分析整个文档的 AI 感，并生成摘要”
```

### 详细示例

```bash
# 改进前后对比
/ai-writing-check --file draft.md
“请检测 AI 感表达，并按以下格式呈现：
- 问题位置（带行号）
- 问题类型和原因
- 具体改进建议
- 改进效果”

# 自动修复模式
/ai-writing-check --file report.md --fix
“请自动修复检测到的模式，并报告结果”

# 项目的 AI 感报告
/ai-writing-check --dir . --severity all
“请分析整个项目的 AI 感：
1. 统计信息（按模式分类的检测数量）
2. 问题最严重的前 5 个文件
3. 改进优先级矩阵
4. 分阶段改进计划”
```

### 高级用法

```bash
# 应用自定义规则
/ai-writing-check --file spec.md
“请根据以下附加标准检查技术规格书：
- 模糊表达（适当的、根据需要）
- 缺乏具体性（高速 → 具体数值）
- 术语使用不一致”

# 用于 CI/CD 集成的检查
/ai-writing-check --dir docs/ --severity high
“请以可在 GitHub Actions 中执行的格式输出结果：
- 错误数量和文件名
- 需要修改的行号
- 设置退出代码”

# 风格指南合规性检查
/ai-writing-check --file manual.md
“请根据公司风格指南进行附加检查：
- 敬语的使用（统一使用“です・ます”体）
- 恰当使用专业术语
- 考虑读者”
```

### 注意事项

- AI 感的判断因上下文而异，请将建议作为参考
- 根据文档类型（技术文档、博客、手册等）调整标准
- 无需接受所有建议，请选择合适的建议
- `--fix` 选项会自动修复检测到的模式

### 命令执行时的行为

执行 `/ai-writing-check` 命令时，Claude 会执行以下操作：

1. **模式检测**：从指定的文件或文本中检测 AI 感模式
2. **具体修改建议**：针对每个问题，提供带行号的修改建议
3. **--fix 模式**：自动修复检测到的模式，并以摘要形式显示结果
4. **报告生成**：提供检测数量、改进优先级和修改前后的对比

Claude 会读取实际文件内容，并根据 textlint-rule-preset-ai-writing 的规则执行分析。

### 参考

此命令参考 [textlint-rule-preset-ai-writing](https://github.com/textlint-ja/textlint-rule-preset-ai-writing) 的规则集创建。这是一个 textlint 规则预设，用于检测 AI 生成文章中的机械模式，并促进更自然的表达。
