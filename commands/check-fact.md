## 事实核查

该命令用于参考项目内的代码库、文档（docs/、README.md 等），以确认所提供信息的准确性。

### 使用方法

```bash
# 基本用法
/check-fact "Flutter 应用中使用了 Riverpod"

# 一次性确认多条信息
/check-fact "该项目使用了 GraphQL，并使用 auto_route 管理路由"

# 确认特定的技术规格
/check-fact "认证使用了 JWT，未使用 Firebase Auth"
```

### 确认流程

1. **信息源的优先级**
   - 代码库（最可靠）
   - README.md、docs/ 内的文档
   - package.json、pubspec.yaml 等配置文件
   - Issue、Pull Request 的讨论历史

2. **判定结果的分类**
   - `✅ 正确` - 信息与代码库完全一致
   - `❌ 错误` - 信息明显错误
   - `⚠️ 部分正确` - 部分准确但不完整
   - `❓ 无法判断` - 缺少确认所需的信息

3. **明确依据**
   - 相关文件名和行号
   - 相关的代码片段
   - 文档中的相关部分

### 报告格式

```
## 事实核查结果

### 验证对象
“[用户提供的信息]”

### 结论
[✅/❌/⚠️/❓] [判定结果]

### 依据
- **文件**: `path/to/file.dart:123`
- **内容**: [相关的代码/文章]
- **补充**: [补充说明]

### 详细说明
[如果是错误，则提供正确的信息]
[如果是部分正确，则指出不准确的部分]
[如果是无法判断，则说明缺少的信息]
```

### 基本示例

```bash
# 确认项目技术栈
/check-fact "该应用采用 Flutter + Riverpod + GraphQL 的架构"

# 确认实现状况
/check-fact "已实现暗黑模式功能，并可从用户设置中切换"

# 确认架构
/check-fact "状态管理全部由 Riverpod 完成，未使用 BLoC"

# 确认安全实现
/check-fact "认证令牌已加密保存在 secure storage 中"
```

### 与 Claude 协作

```bash
# 分析整个代码库后进行确认
ls -la && find . -name "pubspec.yaml" -exec cat {} \;
/check-fact "该项目使用的主要依赖是..."

# 确认特定功能的实现状况
grep -r "authentication" . --include="*.dart"
/check-fact "认证功能为自定义实现，未使用第三方认证"

# 确认与文档的一致性
cat README.md
/check-fact "README 中记载的功能已全部实现"
```

### 应用场景

- 编写技术规格书时：确认记载内容的准确性
- 项目交接时：确认对现有实现的理解
- 向客户报告前：确认实现状况的事实
- 撰写技术博客时：验证文章内容的准确性
- 准备面试/说明材料时：确认项目概要的准确性

### 注意事项

- 代码库是最可靠的信息源
- 如果文档过时，以实现为准
- 如果缺少判断所需的信息，则直接回答“无法判断”
- 对涉及安全的信息尤其要谨慎验证
