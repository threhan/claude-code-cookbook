## 序贯思维

通过动态、迭代的思维过程，分阶段解决复杂问题。这是一种灵活的方法，可以在思考过程中改变方向或进行修正。

### 使用方法

```bash
# 请求 Claude 进行分阶段思考
“请用 sequential-thinking 探讨 [课题]”
```

### 基本示例

```bash
# 算法设计
“请用 sequential-thinking 设计高效的缓存策略”

# 问题解决
“请用 sequential-thinking 解决数据库性能问题”

# 设计探讨
“请用 sequential-thinking 探讨实时通知系统的设计”
```

### 与 Claude 协作

```bash

“请用 sequential-thinking 探讨认证系统的实施方针。考虑 OAuth2、JWT、会话管理”

# Bug 原因分析
“请用 sequential-thinking 分析内存泄漏的原因。包括代码审查和性能分析结果”

cat src/complex_module.js

```

### 思维过程

1.  **初步分析** - 对问题的基本理解和分解
2.  **提出假设** - 提出解决方案的假设
3.  **验证和修正** - 验证假设，并根据需要进行修正
4.  **分支和探索** - 探索多种解决方案路径
5.  **整合和结论** - 得出最佳解决方案

### 特点

-   **动态调整** - 可在思考过程中改变方向
-   **假设验证** - 提出并验证假设的循环
-   **分支思维** - 同时探索多种思维路径
-   **分阶段完善** - 分阶段完善解决方案
-   **灵活性** - 根据新信息调整方针


### 注意事项

sequential-thinking 最适合需要分阶段深化思考的复杂问题。对于简单问题或有明确答案的问题，请使用常规提问方式。

### 执行示例

```bash
# 使用示例
“请用 sequential-thinking 探讨 GraphQL schema 的设计”

# 预期行为
# 1. 初步分析: 分析 GraphQL schema 的基本需求
# 2. 提出假设: 探讨多种设计模式

# 4. 分支: 根据需要探索新方法
# 5. 整合: 提出最佳的 schema 设计
```
