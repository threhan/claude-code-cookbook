## 任务

启动专用代理，自主执行复杂的搜索、调查和分析任务。通过结合多种工具进行大规模信息处理，并重视上下文效率。

### 使用方法

```bash
# 委托 Claude 执行任务
“请用‘任务’调查 [课题]”
```

### 任务的特点

**自主执行**

- 结合多种工具自动执行
- 分阶段收集和分析信息
- 整合结果并提供结构化报告

**高效的信息处理**

- 优化上下文消耗
- 大规模文件搜索和分析
- 从外部信息源收集数据

**质量保证**

- 检查信息源的可靠性
- 从多个角度进行验证
- 自动补充缺失信息

### 与其他命令的区别

#### 任务 vs 其他命令

| 命令 | 主要用途 | 执行方式 | 信息收集 |
|---|---|---|---|
| **任务** | 调查、分析、搜索 | 自主执行 | 多个来源 |
| ultrathink | 深入思考、判断 | 结构化思考 | 以现有知识为中心 |
| sequential-thinking | 解决问题、设计 | 分阶段思考 | 根据需要 |

#### 判断流程图

```
需要收集信息吗？

│          ├─ 是 → **任务**
```

### 适用场景/不适用场景

**适用场景**

- 复杂的代码库调查（依赖关系、架构分析）
- 大规模文件搜索（特定的实现模式、配置文件）
- 收集和整理外部信息（技术趋势、库调查）
- 整合来自多个来源的信息（日志分析、指标分析）
- 重复性调查工作（安全审计、技术债调查）
- 希望避免上下文消耗的大规模分析

**不适用场景**

- 简单问题或可以用现有知识回答的内容
- 短时间内即可完成的单次工作
- 需要交互式确认和咨询的工作
- 实施或设计的判断（plan 或思考类命令更合适）

### 执行流程和质量保证

#### 任务的执行流程

```
1. 初步分析
   ├─ 分解课题并确定调查范围
   ├─ 选择必要的工具和信息源
   └─ 制定执行计划

2. 信息收集
   ├─ 文件搜索/代码分析
   ├─ 收集外部信息
   └─ 数据结构化

3. 分析/整合
   ├─ 分析收集信息的关联性
   ├─ 找出模式或问题点
   └─ 验证假设

4. 报告/建议
   ├─ 结构化结果
   ├─ 创建改进建议
   └─ 提出下一步行动
```

#### 质量保证

- **检查信息源的可靠性**: 通过多个来源核实事实
- **确认覆盖范围**: 检查调查对象是否有遗漏
- **验证一致性**: 核对矛盾信息的一致性
- **评估实用性**: 评估建议的可行性和效果

### 错误处理和限制

#### 常见限制

- **外部 API 的使用限制**: 速率限制或身份验证错误
- **大文件处理限制**: 内存或超时限制
- **访问权限问题**: 对文件或目录的访问限制

#### 错误处理

- **部分结果报告**: 仅使用已获取的信息进行分析
- **提出替代方案**: 在限制条件下提出替代调查方法
- **分阶段执行**: 将大规模任务分阶段执行

### 注意事项

- “任务”最适合复杂且需要自主调查和分析的任务
- 对于简单问题或需要立即回答的情况，请使用常规提问方式
- 调查结果仅供参考，重要决策请务必自行验证
- 收集外部信息时，请注意信息的新旧和准确性
