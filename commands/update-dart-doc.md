## 更新 Dart 文档

系统地管理 Dart 文件的 DartDoc 注释，以维护高质量的中文文档。

### 使用方法

```bash
# 同时执行新增和更新
“为没有 DartDoc 注释的类添加注释，并更新不符合标准的注释”

# 检查 PR 的变更文件
“请检查 PR #4308 中变更文件的 DartDoc 是否有 Claude 标记”

# 整理特定目录的文档
“请为 packages/app/lib/ui/screen/ 目录下的 Widget 类添加 DartDoc”

# 无标记执行
/update-dart-doc --marker false
“改善现有项目的 DartDoc（不添加 Claude 标记）”
```

### 选项

- `--marker <true|false>` : 是否添加 Claude 标记（默认为 true）

### 基本示例

```bash
# 1. 分析目标文件
find . -name "*.dart" -not -path "*/.*" | grep -v "_test.dart" | grep -v "_vrt.dart"
“请识别缺少 DartDoc 的类（注释行数为 0 或少于 30 个字符）”

# 2. 添加文档
“请为已识别的类添加包含必要元素的 DartDoc 注释”

# 3. 确认标记
“请确认所有添加/更新的 DartDoc 中都有 Claude 标记”
```

### 执行步骤

#### 1. 目标要素的优先顺序

1. 🔴 **最优先**: 没有 DartDoc 注释的要素（注释行数为 0）
2. 🟡 **次优先**: 不符合标准的要素（少于 30 个字符或缺少必要要素）
3. 🟢 **确认对象**: 没有 Claude 标记的现有注释

**目标要素**:

- Class（所有类定义）
- Enum（枚举）
- Extension（扩展方法）
- 重要函数（顶层函数，可选）

#### 2. DartDoc 编写规则

**基本结构**:

```dart
/// {要素的概要说明}（30-60 个字符，必需）
///
/// {详细说明}（必须包含作用、使用场景、注意事项，50-200 个字符）
///
/// Generated by Claude 🤖
@注解  // 不更改现有注解
class 类名 {
```

**文章风格**:

- 使用礼貌体（ですます调）: “表示します”“管理するクラスです”
- 标点符号使用“。”“、”
- 中文和半角英数字之间使用半角空格
- 技术术语使用英文: “Authentication 状态”
- 每行不超过 80 个字符

#### 3. 按类别的编写示例

**状态管理类 (Riverpod)**:

```dart
/// 管理水平滑动收拾忽略状态的 State。
///
/// 在需要禁用水平滑动的特定屏幕或操作中使用。
/// 例如，在轮播显示中或特定输入中。
///
/// Generated by Claude 🤖
@Riverpod(keepAlive: true, dependencies: [])
class HorizontalDragGestureIgnoreState extends _$HorizontalDragGestureIgnoreState {
```

**Widget 类**:

```dart
/// 显示用户个人资料的 Widget。
///
/// 垂直排列头像、用户名和状态信息，
/// 点击时跳转到个人资料详情页面。
///
/// Generated by Claude 🤖
class UserProfileWidget extends HookConsumerWidget {
```

#### 4. 现有内容保留规则

1. **如果现有注释符合标准**: 保持原样（不新增）
   - 标准: 30 个字符以上且包含必要要素（概要、详细、标记）
2. **如果现有注释不符合标准**: 完全替换（不重复）
3. **如果没有现有注释**: 添加新注释

**应保留的重要信息**:

- URL 或链接: 以 `See also:` 开头的引用
- TODO 注释: `TODO(user_name):` 格式
- 注意事项: `注意:` 或 `Warning:` 等警告
- 使用示例: 以 `例:` 或 `Example:` 开头的代码
- 技术限制: 性能或限制的描述

### Claude 标记管理

```bash
# 标记格式
/// Generated by Claude 🤖

# 在 PR 的变更文件中确认标记
gh pr diff 4308 --name-only | grep "\.dart$" | xargs grep -l "Generated by Claude"
“请为没有标记的文件添加标记”
```

### 质量检查清单

- ✅ **字符数**: 概要 30-60 字符，详细 50-200 字符，严格遵守
- ✅ **必要要素**: 必须包含概要、详细说明、Claude 标记 3 个要素
- ✅ **完整性**: 记载作用、使用场景、注意事项
- ✅ **一致性**: 文体统一为礼貌体（ですます调）
- ✅ **格式**: 中文和英文之间有半角空格
- ✅ **准确性**: 分析实现，只记载基于事实的描述
- ✅ **结构**: 保留注解，注释放在上方
- ✅ **长度**: 每行不超过 80 个字符
- ✅ **标记**: Claude 的变更必须添加标记

### 注意事项

**🔴 绝对禁止事项**:

- ❌ 更改文档注释以外的代码
- ❌ 对实现细节进行推测（只记载事实）
- ❌ 中英文不自然混用
- ❌ 删除/更改现有注解
- ❌ 与现有注释重复
- ❌ 对测试文件（`*_test.dart`）添加少于字符数标准的注释
- ❌ 对 VRT 文件（`*_vrt.dart`）添加少于字符数标准的注释

**静态分析与提交**:

```bash
# 记录执行结果
ADDED_COMMENTS=0
UPDATED_COMMENTS=0
ERRORS=0

# 变更后确认
melos analyze
if [ $? -ne 0 ]; then
  echo "🔴 错误: 静态分析失败"
  exit 1
fi

# 输出执行摘要
echo "📊 执行结果:"
echo "- 添加的注释: $ADDED_COMMENTS 条"
echo "- 更新的注释: $UPDATED_COMMENTS 条"
echo "- 发生的错误: $ERRORS 条"

# 提交示例
git commit -m "docs: 添加/更新 DartDoc 注释

- 为不符合标准的类、enum、extension 添加 DartDoc
- 根据标准更新少于 30 个字符的注释
- 统一添加 Claude 标记

执行结果:
- 添加: $ADDED_COMMENTS 条
- 更新: $UPDATED_COMMENTS 条

Generated by Claude 🤖"
```

### 执行成功标准

1. **完成判断**: 同时满足以下所有条件时为成功
   - `melos analyze` 通过
   - 发生错误数为 0
   - 添加/更新的注释全部符合标准

2. **部分成功**: 在以下情况下
   - 发生错误数少于 5 条
   - 整体 90% 以上符合标准

3. **失败**: 在以下情况下
   - `melos analyze` 失败
   - 发生错误数 5 条以上
