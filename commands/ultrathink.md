## Ultrathink

针对复杂课题或重要决策，执行分阶段、结构化的思考过程。

### 使用方法

```bash
# 委托 Claude 进行深入思考
“请用 ultrathink 探讨 [课题]”
```

### 基本示例

```bash
# 探讨架构设计
“请用 ultrathink 探讨应该选择微服务还是单体架构”

# 分析技术选型
“请用 ultrathink 分析本项目更适合使用 Rust 还是 TypeScript”

# 深入探讨问题解决方案
“请用 ultrathink 探讨应用程序性能不佳的原因及改进方法”
```

### 与 Claude 协作

```bash
# 业务判断
“请用 ultrathink 探讨新功能的优先级排序。从用户价值、开发成本、技术风险的角度出发”

# 系统设计
“请用 ultrathink 探讨认证系统的设计。考虑安全性、可扩展性、可维护性”

# 权衡分析
“请用 ultrathink 分析选择 GraphQL vs REST API。基于项目需求”

# 重构策略
cat src/legacy_code.js
“请用 ultrathink 为这段遗留代码制定重构策略”
```

### 思考过程

1.  **问题分解** - 将课题分解为构成要素
2.  **MECE 分析** - 无遗漏、无重复地整理
3.  **多角度探讨** - 从技术、业务、用户角度进行分析
4.  **交互式确认** - 在关键决策点与用户确认
5.  **有根据的提案** - 基于数据和逻辑的结论

### 详细示例

```bash
# 解决复杂的技术债务
“请用 ultrathink 探讨一个有 10 年历史的遗留系统的现代化策略。包括分阶段迁移、风险和 ROI”

# 组织性课题
“请用 ultrathink 探讨开发团队的扩展策略。设想从现在的 5 人扩展到 20 人”

# 数据库迁移
“请用 ultrathink 分析从 PostgreSQL 迁移到 DynamoDB。考虑成本、性能、运维方面”
```

### 注意事项

ultrathink 最适合需要花时间深入思考的课题。对于简单问题或需要立即回答的情况，请使用常规提问方式。
