## 更新文档字符串

系统地管理多语言文档字符串/注释，以维护高质量的文档。

### 使用方法

```bash
# 自动检测语言并执行
“为没有文档字符串的类/函数添加，并更新不符合标准的注释”

# 指定语言执行
/update-doc-string --lang python
“请根据 PEP 257 标准更新 Python 文件的文档字符串”

# 整理特定目录的文档
“请为 src/components/ 目录下的函数添加 JSDoc”
```

### 选项

- `--lang <en|zh>`: 文档描述语言（默认为：从现有注释中自动判断，如果没有则为 en）
- `--style <样式>`: 指定文档样式（有特定于语言的默认值）
- `--marker <true|false>`: 是否添加 Claude 标记（默认为：true）

### 基本示例

```bash
# 1. 分析目标文件（自动检测编程语言）
find . -type f \( -name "*.py" -o -name "*.js" -o -name "*.ts" -o -name "*.dart" -o -name "*.go" -o -name "*.rs" \) | grep -v test
“请识别缺少文档字符串的元素（注释行数为 0 或少于 30 个字符）”

# 2. 添加文档（自动判断语言）
“请为已识别的元素添加包含特定语言必要元素的文档字符串”
# → 如果现有注释中有中文，则用中文描述，否则用英文

# 3. 添加文档（明确指定英文）
/update-doc-string --lang en
“Add docstrings with required elements to the identified elements”

# 4. 确认标记
“请确认所有添加/更新的文档字符串都有 Claude 标记”
```

### 执行步骤

#### 1. 目标元素优先级

1. 🔴 **最高优先级**: 没有文档字符串/注释的元素（注释行数为 0）
2. 🟡 **次高优先级**: 不符合标准的元素（少于 30 个字符或缺少必要元素）
3. 🟢 **确认对象**: 没有 Claude 标记的现有注释

**目标元素（通用）**:

- Class/类定义
- Function/函数/方法
- Module/模块 (Python, Go)
- Enum/枚举类型
- Interface/接口 (TypeScript, Go)

#### 3. 现有内容保留规则

1.  **如果现有注释符合标准**: 按原样保留（不添加新的）
    *   标准：30 个字符以上且包含必要元素（摘要、详细信息、标记）
2.  **如果现有注释不符合标准**: 完全替换（不重复）
3.  **如果没有现有注释**: 添加新注释

**应保留的重要信息**:

-   URL 或链接: `See also:`, `@see`, `参照:` 等
-   TODO 注释: `TODO:`, `FIXME:`, `XXX:` 格式
-   注意事项: `Note:`, `Warning:`, `注意:` 等
-   使用示例: `Example:`, `例:`, `# Examples` 等
-   现有的参数/返回值说明

### 质量清单

- ✅ **字符数**: 摘要 30-60 个字符，详细信息 50-200 个字符
- ✅ **必要元素**: 必须包含摘要、详细说明和 Claude 标记这 3 个元素
- ✅ **完整性**: 描述角色、使用上下文和注意事项
- ✅ **语言约定**: 遵守各语言的官方样式指南
- ✅ **异常**: 错误/异常的说明（如果适用）
- ✅ **准确性**: 分析实现，仅根据事实进行描述

### 注意事项

**🔴 绝对禁止**:

- ❌ 更改文档注释以外的代码
- ❌ 对实现细节进行猜测（仅记录事实）
- ❌ 违反语言约定的格式
- ❌ 删除/更改现有的类型注释
- ❌ 与现有注释重复
- ❌ 在测试文件中使用少于字符数标准的注释

**执行和验证**:

```bash
# 记录执行结果
ADDED_COMMENTS=0
UPDATED_COMMENTS=0
ERRORS=0

# 从现有注释中自动判断语言
# 如果检测到中文字符（平假名、片假名、汉字），则为 zh，否则为 en
DOC_LANGUAGE="en"  # 默认
if grep -r '[ぁ-んァ-ヶー一-龠]' --include="*.py" --include="*.js" --include="*.ts" --include="*.dart" --include="*.go" --include="*.rs" . 2>/dev/null | head -n 1; then
  DOC_LANGUAGE="zh"
fi

# 自动检测编程语言并进行静态分析
if [ -f "*.py" ]; then
  pylint --disable=all --enable=missing-docstring .
elif [ -f "*.js" ] || [ -f "*.ts" ]; then
  eslint . --rule 'jsdoc/require-jsdoc: error'
elif [ -f "*.go" ]; then
  golint ./...
elif [ -f "*.rs" ]; then
  cargo doc --no-deps
elif [ -f "*.dart" ]; then
  melos analyze
fi

if [ $? -ne 0 ]; then
  echo "🔴 错误：静态分析失败"
  exit 1
fi

# 输出执行摘要
echo "📊 执行结果:"
echo "- 文档语言: $DOC_LANGUAGE"
echo "- 添加的注释: $ADDED_COMMENTS 条"
echo "- 更新的注释: $UPDATED_COMMENTS 条"
echo "- 错误数: $ERRORS 条"
```

### 执行成功标准

1.  **完成判断**: 满足以下所有条件时成功
    *   特定于语言的静态分析通过
    *   错误数为 0
    *   所有添加/更新的注释都符合标准

2.  **部分成功**: 在以下情况下
    *   错误数少于 5
    *   90% 以上符合标准

3.  **失败**: 在以下情况下
    *   静态分析失败
    *   错误数大于等于 5

### 与 Claude 协作

```bash
# 分析整个项目（自动判断语言）
find . -type f \( -name "*.py" -o -name "*.js" -o -name "*.ts" \)
/update-doc-string
“请根据各语言的最佳实践更新此项目的文档字符串”
# → 如果现有注释中有中文，则以 zh 执行，否则以 en 执行

# 明确以英文文档执行
/update-doc-string --lang en
"Update docstrings following language-specific best practices"

# 明确以中文文档执行
/update-doc-string --lang zh
“请根据各语言的最佳实践更新此项目的文档字符串”

# 无标记执行（自动判断语言）
/update-doc-string --marker false
"Improve existing docstrings without adding Claude markers"

# 英文文档，无标记
/update-doc-string --lang en --marker false
"Improve existing docstrings without adding Claude markers"
```
